Interpretation

BB-001 is a harness test, not a leaderboard final.

Because the code change is trivial, pass/fail alone is useless. A serious benchmark records the cost of arriving at the patch: how many tool calls, whether the agent proved the bug red first, whether it ran smoke tests, how much context it burned, and whether telemetry can be independently reconstructed.

What the run proves

  • Each agent ran in a separate copied workspace.
  • Baseline was captured red before agents touched code.
  • Hermes verified every final workspace independently.
  • Raw logs and diffs were committed, not summarized from memory.
  • Token/cost gaps are marked unavailable instead of invented.

What it does not prove

  • It does not rank deep coding ability.
  • It does not measure long-horizon planning.
  • It does not cover UI, architecture, flaky tests, or repo-scale refactors.
  • It does not treat subscription-plan $0 as free compute.
